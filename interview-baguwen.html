软件测试的完整流程是什么？

需求评审、测试范围拆分、测试计划、环境准备、用例设计、用例评审、测试执行、缺陷提交与跟踪、回归测试、上线验收、测试报告、线上问题复盘。回答时最好补一句：我会在需求阶段提前识别权限、数据、接口、异常流和上线风险。

测试环境准备要检查什么？

版本包、配置文件、数据库脚本、测试账号、角色权限、基础数据、第三方接口地址、缓存和定时任务、日志权限、浏览器或设备范围。环境问题要先和开发、运维确认，避免把环境异常当成业务缺陷。

等价类、边界值、场景法怎么用？

等价类用于减少重复输入组合，边界值用于覆盖临界数据，场景法用于覆盖真实业务路径。比如金额字段要测 0、最小值、最大值、超限、小数精度、负数、空值和非法字符；订单流程要测正常支付、取消、退款、重复回调和库存回滚。

接口测试重点关注哪些点？

请求方法、URL、Header、Token、必填参数、参数类型、边界值、业务状态、权限、幂等、返回码、返回字段、数据库落库、日志、上下游同步和异常提示。做项目表达时要结合 Postman、JMeter、Fiddler、MySQL 去说。

SQL 在测试中怎么用？

常用于准备测试数据、验证落库、核对状态流转、排查缺陷、统计异常数据。常见能力包括 select、where、order by、group by、count、join、update 前备份数据。面试里不要只说会查库，要说能验证业务数据一致性。

Linux 常用命令怎么讲？

查看日志：tail -f、grep；查看进程：ps；查看端口：netstat 或 lsof；查看文件：cat、less；传输文件：scp；权限：chmod；目录：cd、ls、pwd。结合场景讲：接口报 500 时，我会看请求参数、服务日志、异常栈和数据库状态。

Java 基础对测试有什么帮助？

不要求像开发一样深入，但理解接口、对象、异常、集合、日志、数据库连接、HTTP 请求、JSON 序列化，有助于读懂报错、定位接口问题和写自动化脚本。能看懂简单代码逻辑，会让你在缺陷沟通时更有底气。

性能测试常见指标有哪些？

响应时间、吞吐量 TPS/QPS、并发用户数、错误率、CPU、内存、磁盘 IO、数据库慢 SQL、接口超时。回答时说明先做单接口压测，再做业务链路压测，最后结合监控和日志定位瓶颈。

上线前你会重点验收什么？

核心主流程、严重缺陷清零、阻塞问题关闭、回归用例通过、配置和脚本确认、接口连通、权限验证、数据迁移或兼容检查、回滚方案、测试报告和遗留风险说明。上线不是点按钮，是质量风险确认。
